# Plan: Admin Assign Location to Users

## Overview
Allow admin to assign/remove locations to/from users (except themselves) via a dialog triggered from the users table.

## API Endpoint
- **PATCH** `http://localhost:3001/api/user/:id/location`
- **Body**: `{ "location_id": "uuid" }` or `{ "location_id": null }` to remove
- Admin only, cannot assign central location

## Changes Required

### 1. `lib/api-routes.ts`
Add location assignment route:
```typescript
USERS: {
  ...
  ASSIGN_LOCATION: (id: string) => `${BASE_URL}/api/user/${id}/location`,
}
```

### 2. `lib/api.ts`
- Update `User` interface to include optional `location` field:
  ```typescript
  location?: {
    id: string;
    address_line_1: string;
    city: string;
    pincode: string;
    state: string;
    country: string;
  } | null;
  ```
- Add `assignLocation` method to `usersApi`:
  ```typescript
  assignLocation: (userId: string, locationId: string | null, token?: string) =>
    apiRequest(API_ROUTES.USERS.ASSIGN_LOCATION(userId), {
      method: "PATCH",
      body: JSON.stringify({ location_id: locationId }),
      token,
    }),
  ```

### 3. `components/dialogs/assign-location-dialog.tsx` (NEW)
Dialog component with:
- Shows user name at top
- Dropdown/select to choose from available locations
- Option to "Remove Location" if user already has one
- Calls `usersApi.assignLocation()` on submit
- Uses existing `useLocations` hook (fetch ALL locations, not paginated - need to add a `getAllNoPagination` or pass high limit)

### 4. `hooks/use-locations.ts`
Add option to fetch all locations without pagination (for the dropdown):
- Add `fetchAll` method or modify hook to accept `fetchAll: true` option
- Or create a simple `useAllLocations` hook

### 5. `app/(dashboard)/dashboard/users/page.tsx`
- Add "Location" column showing assigned location (city + state) or "Not assigned"
- Add "Actions" column with MapPin icon button to open assign dialog
- Filter out current admin user from actions (cannot assign to self)

### 6. `app/(dashboard)/dashboard/users/students/page.tsx`
Same changes as users page

### 7. `app/(dashboard)/dashboard/users/teachers/page.tsx`
Same changes as users page
