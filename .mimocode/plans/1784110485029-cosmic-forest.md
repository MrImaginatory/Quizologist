# Plan: Admin Tests Page

## Overview
Admin-only page to view all tests across all students with filters and pagination.

## API Endpoint
**GET** `/api/test/all` (Admin only)

**Query Params:**
| Param | Type | Description |
|-------|------|-------------|
| page | number | Default 1 |
| limit | number | Default 10, max 100 |
| status | string | `pending`, `in_progress`, `completed`, `abandoned` |
| subjectId | string | UUID filter |
| dateFrom | string | ISO date string |
| dateTo | string | ISO date string |

**Response:** Same structure as `/history` — `{ tests: TestHistory[], pagination }`

## Changes Required

### 1. `lib/api-routes.ts`
Add route for all tests:
```typescript
TESTS: {
  ...
  ALL: `${BASE_URL}/api/test/all`,
}
```

### 2. `lib/api.ts`
Add `getAll` method to `testsApi`:
```typescript
getAll: (params: { page?: number; limit?: number; status?: string; subjectId?: string; dateFrom?: string; dateTo?: string }, token?: string) =>
  apiRequest<TestHistoryResponse>(url, { token }),
```

### 3. `hooks/use-all-tests.ts` (NEW)
Hook to fetch all tests with filters and pagination:
- Accepts filter params (status, subjectId, dateFrom, dateTo)
- Returns tests, pagination, isLoading, error, refetch

### 4. `components/filters/test-filters.tsx` (NEW)
Filter bar with:
- Status dropdown (All, Pending, In Progress, Completed, Abandoned)
- Date From picker
- Date To picker
- Clear filters button

### 5. `app/(dashboard)/dashboard/tests/page.tsx` (NEW)
Main admin tests page:
- Header: "All Tests" with description
- Filters bar
- DataTable with columns:
  - # (row number)
  - Test ID (monospace font)
  - Status (badge with colors)
  - Score (colored based on value)
  - Correct/Total
  - Date (formatted)
  - Actions (View Result button for completed tests)
