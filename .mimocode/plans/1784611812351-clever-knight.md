# Bulk Import Redesign: Support 10,000 Rows

## Problem

The current bulk import system fails at scale because:
1. **Express body size limit**: Default `express.json()` is 100KB — a 10,000-question payload is several MB
2. **No chunking**: Frontend sends ALL questions in one HTTP request
3. **Sequential inserts**: Backend calls `Question.create()` one-by-one (500 individual INSERTs for 500 questions)
4. **No transactions**: Partial failures leave orphaned data
5. **In-memory duplicate check**: Loads ALL existing questions into a JS `Set` — won't scale
6. **Zod `.max(500)`**: Hard cap at 500 questions in validation
7. **No progress feedback**: User sees nothing during a potentially minutes-long import

## Solution: Chunked Upload + Batch DB Inserts

### Architecture

```
Frontend (chunked)           Backend (batched)
─────────────────           ─────────────────
Excel → parse → resolve     POST /api/question/bulk
         │                         │
    Send batch 1 (500)  ──────►  Validate → bulkCreate(500)
    ◄──── result 1 ─────         │
    Send batch 2 (500)  ──────►  Validate → bulkCreate(500)
    ◄──── result 2 ─────         │
    ...                          ...
    Send batch N (≤500)  ──────►  Validate → bulkCreate(N)
    ◄──── result N ─────         │
         │                         │
    Merge all results       Return per-batch summary
```

### 3. Backend: Rewrite `bulkCreate` Service Method

**File**: `backend/questionService/src/modules/question/questionImport.service.ts`

Replace the sequential loop with:

```ts
static async bulkCreate(
  questions: BulkQuestionInput[],
  defaultUserId: string,
  user?: { userId: string; role: string }
): Promise<BulkImportResult> {
  const BATCH_SIZE = 500;
  const errors: { row: number; reason: string }[] = [];
  let imported = 0;

  // 1. Teacher authorization check (once)
  let allowedPairs: Set<string> | null = null;
  if (user?.role === "teacher") {
    const assignments = await TeacherAssignment.findAll({
      where: { teacher_id: user.userId },
      attributes: ["course_id", "subject_id"],
      raw: true,
    });
    allowedPairs = new Set(
      assignments.map((a) => `${a.course_id}|${a.subject_id || ""}`)
    );
  }

  // 2. Pre-validate all rows (fast, no DB writes)
  const validRows: { index: number; data: any; dedupeKey: string }[] = [];
  for (let i = 0; i < questions.length; i++) {
    const row = i + 2;
    const q = questions[i];

    // Teacher auth
    if (allowedPairs) {
      const pairKey = `${q.course_id}|${q.subject_id || ""}`;
      if (!allowedPairs.has(pairKey)) {
        errors.push({ row, reason: "Not assigned to this course/subject" });
        continue;
      }
    }

    // Validate choices
    const validChoices = q.choices.filter((c) => c?.trim());
    if (validChoices.length < 2) {
      errors.push({ row, reason: "At least 2 valid options required" });
      continue;
    }

    // Validate correct answer
    if (!validChoices.includes(q.correctAnswer)) {
      errors.push({ row, reason: "Correct answer doesn't match any option" });
      continue;
    }

    validRows.push({
      index: i,
      data: {
        type: "mcq",
        question: q.question,
        choices: validChoices,
        correctAnswer: q.correctAnswer,
        explanation: q.explanation || null,
        videoUrl: q.videoUrl || null,
        difficulty: q.difficulty || "normal",
        topic_id: q.topic_id,
        subject_id: q.subject_id,
        course_id: q.course_id,
        questionAddedBy: q.questionAddedBy || defaultUserId,
      },
      dedupeKey: `${q.question.toLowerCase()}|${q.topic_id}`,
    });
  }

  // 3. Batch duplicate check using DB (not in-memory)
  //    For each batch, query existing questions for the topic_ids in that batch
  const batchTopicIds = [...new Set(validRows.map((r) => r.data.topic_id))];
  const existingQuestions = await Question.findAll({
    where: { topic_id: { [Op.in]: batchTopicIds }, deletedAt: null },
    attributes: ["question", "topic_id"],
  });
  const existingSet = new Set(
    existingQuestions.map((q) => `${q.question.toLowerCase()}|${q.topic_id}`)
  );

  // 4. Process in batches using Sequelize bulkCreate
  for (let i = 0; i < validRows.length; i += BATCH_SIZE) {
    const batch = validRows.slice(i, i + BATCH_SIZE);
    const batchRecords: any[] = [];
    const batchErrors: { row: number; reason: string }[] = [];

    for (const item of batch) {
      const row = item.index + 2;
      if (existingSet.has(item.dedupeKey)) {
        batchErrors.push({ row, reason: "Duplicate question in this topic" });
        continue;
      }
      batchRecords.push(item.data);
      existingSet.add(item.dedupeKey); // catch intra-batch dupes
    }

    if (batchRecords.length > 0) {
      try {
        const result = await Question.bulkCreate(batchRecords, {
          validate: true,
          returning: false,
        });
        imported += result.length;
      } catch (err: any) {
        // If bulkCreate fails, fall back to individual inserts to get per-row errors
        for (const record of batchRecords) {
          try {
            await Question.create(record);
            imported++;
          } catch (innerErr: any) {
            const originalIndex = validRows.findIndex(
              (r) => r.data.question === record.question && r.data.topic_id === record.topic_id
            );
            batchErrors.push({
              row: originalIndex + 2,
              reason: innerErr.message || "Failed to create",
            });
          }
        }
      }
    }

    errors.push(...batchErrors);
  }

  return {
    totalRows: questions.length,
    imported,
    failed: errors.length,
    errors,
  };
}
```

### 4. Frontend: Add Chunked Upload with Progress

**File**: `frontend/app/(dashboard)/dashboard/questions/import/page.tsx`

Replace the single `handleImport` with chunked upload:

```ts
const BATCH_SIZE = 500;
const [importProgress, setImportProgress] = useState({ current: 0, total: 0 });

const handleImport = async () => {
  setIsLoading(true);
  const readyQuestions = parsedQuestions
    .filter((q) => q.status === "ready")
    .map((q) => ({
      type: q.type as "mcq",
      question: q.question,
      choices: q.choices,
      correctAnswer: q.correctAnswer,
      explanation: q.explanation || "",
      videoUrl: q.videoUrl || "",
      difficulty: "normal",
      topic_id: q.topic_id,
      subject_id: q.subject_id,
      course_id: q.course_id,
      questionAddedBy: "",
    }));

  const totalBatches = Math.ceil(readyQuestions.length / BATCH_SIZE);
  setImportProgress({ current: 0, total: readyQuestions.length });

  let totalImported = 0;
  let totalFailed = 0;
  const allErrors: { row: number; reason: string }[] = [];

  try {
    for (let i = 0; i < readyQuestions.length; i += BATCH_SIZE) {
      const batch = readyQuestions.slice(i, i + BATCH_SIZE);
      const batchNumber = Math.floor(i / BATCH_SIZE) + 1;

      const result = await questionsApi.bulkImport(batch, token || undefined);

      totalImported += result.imported;
      totalFailed += result.failed;
      allErrors.push(...(result.errors || []));

      setImportProgress({
        current: Math.min(i + BATCH_SIZE, readyQuestions.length),
        total: readyQuestions.length,
      });
    }

    setImportResult({
      totalRows: parsedQuestions.length,
      imported: totalImported,
      failed: totalFailed,
      errors: allErrors,
    });
    setStep("result");
  } catch (error) {
    console.error("Import failed:", error);
  } finally {
    setIsLoading(false);
  }
};
```

## Verification

1. **Small import (10 questions)**: Should work as before, no regressions
2. **Medium import (500 questions)**: Should complete in <30 seconds with progress bar
3. **Large import (5,000 questions)**: Should complete in <2 minutes with progress updates
4. **Max import (10,000 questions)**: Should complete without timeout or memory error
5. **Duplicate detection**: Questions with same text + topic should be flagged as errors
6. **Teacher restrictions**: Teachers should only import to assigned courses/subjects
7. **Error handling**: Invalid rows should be skipped with clear error messages, valid rows should still be imported
8. **Build check**: `pnpm build` in both frontend and backend should pass
