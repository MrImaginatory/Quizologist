# Plan: Restrict Teacher Test View to Assigned Courses/Subjects

## Problem Statement

Two issues exist with the teacher's test management view:

### Issue 1: Teacher sees ALL test results (not just assigned courses/subjects)
- The `GET /test/student/:studentId/results` endpoint returns **all** completed tests for a student, regardless of course/subject
- A teacher can view test results for courses they don't teach
- The backend has no assignment-based filtering on this endpoint

### Issue 2: Course/Subject dropdowns show ALL courses (not teacher's assigned ones)
- The tests page loads `useCourses()` and `useSubjects()` which fetch ALL courses/subjects from the Content Service
- The dropdowns should only show courses the teacher is assigned to

## Root Cause Analysis

### Issue 1 Root Cause
**File:** `backend/testService/src/modules/testSession/testSession.service.ts:476-578`

The `getStudentResults()` method only checks:
```typescript
if (requesterRole === "student" && requesterId !== studentId) {
  throw ApiError.forbidden("You can only view your own results");
}
```
Teachers and admins bypass this check and can view ANY student's results. No course/subject filtering is applied.

### Issue 2 Root Cause
**File:** `frontend/app/(dashboard)/dashboard/tests/page.tsx:58-59`

```typescript
const { courses } = useCourses({ limit: 100 });     // ALL courses
const { subjects } = useSubjects({ limit: 100 });    // ALL subjects
```

These hooks fetch from the Content Service without any teacher-scoping. The teacher's assigned courses/subjects are available via `GET /teacher/teacher/:teacherId` but are not used for the dropdowns.

## Implementation Plan

### Step 1: Create a Backend Endpoint for Teacher's Assigned Courses/Subjects

**File to create/modify:** `backend/teacherService/src/modules/teacherAssignment/teacherAssignment.service.ts`

Add a new method `getTeacherAssignedCoursesAndSubjects(teacherId)` that returns the courses and subjects the teacher is assigned to, derived from `TeacherAssignment` records.

**File to modify:** `backend/teacherService/src/modules/teacherAssignment/teacherAssignment.routes.ts`

Add route: `GET /teacher/teaching/courses-and-subjects`

**File to modify:** `backend/teacherService/src/modules/teacherAssignment/teacherAssignment.controller.ts`

Add controller method for the new route.

**File to modify:** `backend/apiGateway/src/config/routes.ts`

Add gateway route: `/teacher/teaching/courses-and-subjects` → teacher service

### Step 2: Frontend — Use Teacher's Assigned Courses/Subjects for Dropdowns

**File to modify:** `frontend/app/(dashboard)/dashboard/tests/page.tsx`

Replace:
```typescript
const { courses } = useCourses({ limit: 100 });
const { subjects } = useSubjects({ limit: 100 });
```

With a call to the new endpoint for teachers:
```typescript
if (isTeacher) {
  // Use teacher's assigned courses/subjects
} else {
  // Use all courses/subjects (admin)
}
```

**File to modify:** `frontend/lib/api.ts`

Add API method: `teachersApi.getTeachingCoursesAndSubjects()`

**File to modify:** `frontend/lib/api-routes.ts`

Add route constant: `TEACHING_COURSES_AND_SUBJECTS`

### Step 3: Backend — Filter Test Results by Teacher's Assignments

**File to modify:** `backend/testService/src/modules/testSession/testSession.service.ts`

In `getStudentResults()` (lines 476-578), add teacher assignment filtering:

1. If `requesterRole === "teacher"`:
   - Fetch the teacher's `TeacherAssignment` records
   - Determine which course/subject pairs the teacher is assigned to
   - Filter test sessions to only include tests where the student has enrollments matching the teacher's assigned courses/subjects

2. Same logic for `getStudentResultSummary()` (lines 580-661)

3. Same logic for `getStudentPerformance()` (lines 435-474)

**Implementation approach:**
```typescript
if (requesterRole === "teacher") {
  // 1. Get teacher's assignments from TeacherAssignment table
  // 2. Get student's enrollments matching those assignments
  // 3. Filter test sessions to only those related to enrolled courses/subjects
  // Since test_sessions don't directly store course_id, we need to check
  // test_selections which DO have course_id/subject_id/topic_id
}
```

**Key consideration:** Test sessions have `test_selections` that store the `course_id`, `subject_id`, `topic_id` for each test. We can join through `test_selections` to filter by course/subject.

### Step 4: Backend — Filter Test Detail by Teacher's Assignments

**File to modify:** `backend/testService/src/modules/testSession/testSession.service.ts`

In `getTestDetailForAdmin()` (lines 374-433), add teacher assignment check:
- If role is teacher, verify the test session's selections match the teacher's assigned courses/subjects

## Verification

1. **Teacher dropdown test:** Login as teacher → Tests page → Course dropdown should only show courses the teacher is assigned to
2. **Subject cascade test:** Select a course → Subject dropdown should only show subjects under that course that the teacher is assigned to
3. **Test results filtering test:** Teacher should only see test results for students enrolled in their assigned courses/subjects
4. **Admin unaffected test:** Login as admin → All courses/subjects shown → All test results visible
5. **Cross-teacher isolation test:** Teacher A should not see test results from Teacher B's courses
