# Plan: Fix Predefined Test Attempt Tracking

## Problem Statement

A student completed a predefined test (max_attempts=1) but the "Available Tests" page still shows "Start Test" button enabled. The test should show as "Completed" instead.

## Root Cause

Three gaps across backend and frontend:

### Gap 1: Backend `getPendingTests` incomplete attempt check
**File:** `testService/src/modules/predefinedTest/predefinedTest.service.ts:367`

Current logic:
```typescript
if (assignedStudent.status === "completed" && test.max_attempts <= 1) {
  continue; // Skip
}
```

**Missing:** Does NOT check `status === "started"` (in-progress). Also doesn't handle the case where `max_attempts > 1` but student already used all attempts.

### Gap 2: Frontend ignores `student_status`
**File:** `frontend/hooks/use-pending-tests.ts:7-18`

The `PendingTest` interface doesn't include `student_status` field, even though the backend returns it.

### Gap 3: Frontend has no "completed" status variant
**File:** `frontend/app/(dashboard)/student/available-tests/page.tsx:22,44-52`

`TestStatus` type only has `"available" | "upcoming" | "expired"`. No way to show "Completed" or disable the button.

## Implementation Plan

### Step 1: Fix backend `getPendingTests` attempt check

**File:** `backend/testService/src/modules/predefinedTest/predefinedTest.service.ts`

Replace the simple status check with a proper attempt count (matching `startTest` logic):

```typescript
// Count actual attempts
const existingStudentTests = await PredefinedTestStudent.findAll({
  where: { predefined_test_id: test.id, student_id: studentId },
});
const sessionCount = await TestSession.count({
  where: {
    predefined_test_id: test.id,
    student_id: studentId,
    status: { [Op.in]: ["completed", "abandoned"] },
  },
});
const totalAttempts = Math.max(existingStudentTests.length, sessionCount);

if (totalAttempts >= test.max_attempts) {
  continue; // No attempts remaining
}

// Also skip if student has an in-progress session
const hasInProgress = existingStudentTests.some(s => s.status === "started");
if (hasInProgress) {
  continue; // Already taking this test
}
```

### Step 2: Add `student_status` to frontend `PendingTest` interface

**File:** `frontend/hooks/use-pending-tests.ts`

Add `student_status` field:
```typescript
interface PendingTest {
  // ... existing fields
  student_status?: "assigned" | "started" | "completed";
}
```

### Step 3: Add "completed" status to frontend

**File:** `frontend/app/(dashboard)/student/available-tests/page.tsx`

1. Add `"completed"` to `TestStatus` type:
```typescript
type TestStatus = "available" | "upcoming" | "expired" | "completed";
```

2. Update `getTestStatus` to check `student_status`:
```typescript
const getTestStatus = (test: any): TestStatus => {
  if (test.student_status === "completed") return "completed";
  if (test.status === "upcoming") return "upcoming";
  if (test.is_scheduled && test.end_time) {
    const now = new Date();
    const endTime = new Date(test.end_time);
    if (now > endTime) return "expired";
  }
  return "available";
};
```

3. Update badge and button rendering:
- Badge: Show "Completed" in green for completed tests
- Button: Disable and show "Completed" text for completed tests
- Summary cards: Count completed tests separately

### Step 4: Add "completed" status to join test page (if applicable)

Same logic â€” check `student_status` and show appropriate UI.
