# Plan: Admin Analytics Dashboard

## Overview

Build a comprehensive analytics dashboard for the admin panel with 4 analytics modules, global filters, interactive charts, and a scalable layout.

---

## Architecture

### Backend
- **New endpoints** in `dashboardService` (read-only, Sequelize ORM queries — no raw SQL)
- **No new tables** — all data derived from existing tables (users, questions, test_sessions, enrollments, teacher_assignments, locations, subjects, topics)
- **Models needed:** User, Location, TestSession, TestAnswer, Question, Topic, Subject, Course, Enrollment, TeacherAssignment

### Frontend
- **New page:** `/dashboard/analytics`
- **New hook:** `useAdminAnalytics.ts`
- **New chart components** for admin-specific visualizations
- **New sidebar entry** for "Analytics"

---

## Backend: New API Endpoints

All queries use **Sequelize ORM** (models + `findAll` with `include`, `attributes`, `group`, `fn`, `col`, `literal`). No raw SQL.

### Endpoint 1: `GET /api/dashboard/analytics/teacher-student-ratio`

**ORM Approach:**
1. Query teachers grouped by `location_id`: `User.findAll({ where: { role: 'teacher', deletedAt: null }, attributes: ['location_id', [fn('COUNT', col('id')), 'count']], group: ['location_id'], raw: true })`
2. Query students grouped by `location_id`: same pattern with `role: 'student'`
3. Query all locations: `Location.findAll({ where: { deletedAt: null }, attributes: ['id', 'city', 'state'] })`
4. Combine in JavaScript to build ratio data per location

**Response:**
```json
{
  "data": {
    "locations": [
      { "id": "uuid", "city": "Mumbai", "state": "Maharashtra", "teacher_count": 5, "student_count": 45, "ratio": "1:9" },
      ...
    ],
    "total_teachers": 11,
    "total_students": 100
  }
}
```

**Filters:** `location_id` (optional)

### Endpoint 2: `GET /api/dashboard/analytics/top-students-by-location`

**ORM Approach:**
1. `TestSession.findAll({ where: { status: 'completed', ...dateFilters }, attributes: ['student_id', [fn('AVG', col('score')), 'avg_score'], [fn('COUNT', col('id')), 'total_tests']], group: ['student_id'], order: [[literal('avg_score'), 'DESC']], limit })`
2. Extract `studentIds` from results
3. `User.findAll({ where: { id: { [Op.in]: studentIds }, role: 'student', deletedAt: null, ...locationFilter }, include: [{ model: Location, attributes: ['city', 'state'] }], attributes: { exclude: ['password'] } })`
4. Merge and rank in JavaScript

**Response:**
```json
{
  "data": {
    "students": [
      { "id": "uuid", "fname": "John", "lname": "Doe", "email": "john@doe.com",
        "city": "Mumbai", "total_tests": 10, "avg_score": 85.5, "rank": 1 },
      ...
    ]
  }
}
```

**Filters:** `location_id`, `date_from`, `date_to`, `limit` (default 10)

### Endpoint 3: `GET /api/dashboard/analytics/least-questions`

**ORM Approach:**
1. `Topic.findAll({ include: [{ model: Subject, as: 'subject', attributes: ['id', 'name'], include: [{ model: Course, as: 'course', attributes: ['id', 'name'] }] }], attributes: { include: [[fn('COUNT', col('Questions.id')), 'question_count']] }, group: ['Topic.id', 'Subject.id', 'Course.id'], order: [[literal('question_count'), 'ASC']], limit: 10, subQuery: false })`
2. Add status label in JavaScript: `questionCount < 5 ? 'needs_questions' : 'adequate'`

**Response:**
```json
{
  "data": {
    "topics": [
      { "topicId": "uuid", "topicName": "Binary Trees", "subjectName": "Data Structures",
        "courseName": "Computer Science", "questionCount": 2, "status": "needs_questions" },
      ...
    ]
  }
}
```

**Filters:** `course_id`, `subject_id` (optional)

### Endpoint 4: `GET /api/dashboard/analytics/subjects-needing-attention`

**ORM Approach:**
1. Get completed test sessions: `TestSession.findAll({ where: { status: 'completed', ...dateFilters }, attributes: ['id', 'student_id', 'score'] })`
2. Get test answers with questions: `TestAnswer.findAll({ where: { test_session_id: { [Op.in]: sessionIds } }, include: [{ model: Question, attributes: ['subject_id'] }] })`
3. Group by subject, calculate avg score and below-passing count in JavaScript
4. For each weak subject, query low-performing students separately

**Response:**
```json
{
  "data": {
    "subjects": [
      { "subjectId": "uuid", "subjectName": "Data Structures", "courseName": "CS",
        "avgScore": 42.5, "studentCount": 30, "belowPassingCount": 18,
        "lowPerformers": [
          { "studentId": "uuid", "fname": "Jane", "lname": "Doe", "avgScore": 35.2 }
        ]
      },
      ...
    ]
  }
}
```

**Filters:** `date_from`, `date_to`
