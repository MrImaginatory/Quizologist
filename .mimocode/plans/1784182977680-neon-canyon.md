# Predefined Test Validation Rules - Implementation Plan

## Overview

Add validation rules to prevent data integrity issues:
1. Cannot edit questions if any user has started/completed the test
2. Can only add students if more than X minutes before test starts (configurable)
3. Cannot deactivate if any student has started or test is starting soon

---

### New Environment Variables

Add to `.env` files:

## Validation Rules

### Rule 1: Cannot Edit Questions if Test is In Progress

**Condition:** If any student has started (status="started") or completed (status="completed") the test in `predefined_test_students`, or if any `test_sessions` exist for this test, then questions cannot be edited.

**Error Message:** "Cannot edit questions: students have already started or completed this test"

**Implementation:**
- Check `PredefinedTestStudent` for any records with status "started" or "completed"
- Check `TestSession` for any records with `predefined_test_id` matching
- If either exists, throw error

### Rule 2: Can Only Add Students if Test Starts in > X Minutes

**Condition:** If the test is scheduled and starts within `PREDEFINED_TEST_MIN_STUDENT_ASSIGN_MINUTES` minutes, student assignment is not allowed.

**Error Message:** "Cannot assign students: test starts within {X} minutes"

**Implementation:**
- Check if test is scheduled
- If scheduled, check if `start_time - now < PREDEFINED_TEST_MIN_STUDENT_ASSIGN_MINUTES`
- If true, throw error

### Rule 3: Cannot Deactivate if Test is In Progress or Starting Soon

**Conditions:**
1. If any student has started the test (status="started" or "completed") → cannot deactivate
2. If test is scheduled and starts within `PREDEFINED_TEST_MIN_DEACTIVATE_MINUTES` minutes → cannot deactivate

**Error Messages:**
- "Cannot deactivate: students have already started this test"
- "Cannot deactivate: test starts within {X} minutes"

**Implementation:**
- Check `PredefinedTestStudent` for any records with status "started" or "completed"
- Check `TestSession` for any records with `predefined_test_id` matching
- If test is scheduled, check if `start_time - now < PREDEFINED_TEST_MIN_DEACTIVATE_MINUTES`
- If any condition is true, throw error

### Helper Function: `checkTestActivity()`

```typescript
async function checkTestActivity(predefinedTestId: string): Promise<boolean> {
  // Check predefined_test_students for started/completed status
  const studentActivity = await PredefinedTestStudent.findOne({
    where: {
      predefined_test_id: predefinedTestId,
      status: { [Op.in]: ["started", "completed"] },
    },
  });

  if (studentActivity) return true;

  // Check test_sessions for this predefined test
  const TestSession = require("../testSession/testSession.model").default;
  const sessionActivity = await TestSession.findOne({
    where: {
      predefined_test_id: predefinedTestId,
      status: { [Op.in]: ["in_progress", "completed", "abandoned"] },
    },
  });

  return !!sessionActivity;
}
```
