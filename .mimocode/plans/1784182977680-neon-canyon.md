# Structured Logging System - Implementation Plan

## Overview

Add a structured logging system to all 8 backend services using **winston**. Each phase completes one service. After each phase, I will prompt you to verify before moving to the next.

---

## Shared Logger Setup (Before Phase 1)

Create `backend/shared/logger/` package first — this is used by all services.

### Files to Create
| File | Purpose |
|------|---------|
| `backend/shared/logger/logger.ts` | Winston logger factory |
| `backend/shared/logger/requestLogger.ts` | Express request logging middleware |
| `backend/shared/logger/index.ts` | Export barrel |

### Log Features
- Structured JSON logs to files
- Colored text logs to console
- Request logging: method, endpoint, status, duration, IP
- Error logging with stack traces
- Log rotation: 5MB max, 5 files per service

## Phase 1: API Gateway

**Service:** `backend/apiGateway/`
**Port:** 3000
**Log file:** `backend/logs/api-gateway.log`

1. Add winston to `package.json`
2. Create logger instance in `index.ts`
3. Add `requestLogger` middleware before routes
4. Replace `console.log`/`console.error` with logger
5. Log server startup
