# Student Enrollment Filtered Data - Implementation Plan

## Problem

The `StartTestDialog` fetches ALL subjects/topics for a course, but the student is only enrolled in specific subjects. This causes:
- Student sees all 10 subjects instead of their 3 enrolled subjects
- Student sees all topics instead of topics for enrolled subjects

## Solution

Add two new endpoints to the student service that return filtered enrollment data:

1. `GET /api/enrollment/subjects?course_id=uuid` — Get enrolled subjects for a course
2. `GET /api/enrollment/topics?course_id=uuid&subject_id=uuid` — Get enrolled topics for a subject

---

## Backend Changes

### 1. New Endpoint: `GET /api/enrollment/subjects`

Returns only the subjects the student is enrolled in for a specific course.

**Query Params:**
| Param | Type | Required | Description |
|-------|------|----------|-------------|
| course_id | string | Yes | Course UUID |

**Logic:**
1. Get student ID from JWT
2. Query enrollments where `student_id` = current user AND `course_id` = provided
3. If enrollment has `subject_id` → return that subject
4. If enrollment has `subject_id = null` → return ALL subjects for the course
5. Deduplicate results

**Response:**
```json
{
  "subjects": [
    { "id": "uuid", "name": "Intellectual Property Rights" },
    { "id": "uuid", "name": "Cyber Law And Information Technology Act" },
    { "id": "uuid", "name": "Insolvency And Bankruptcy Code" }
  ]
}
```

### 2. New Endpoint: `GET /api/enrollment/topics`

Returns only the topics the student is enrolled in for a specific subject.

**Query Params:**
| Param | Type | Required | Description |
|-------|------|----------|-------------|
| course_id | string | Yes | Course UUID |
| subject_id | string | Yes | Subject UUID |

**Logic:**
1. Get student ID from JWT
2. Query enrollments where `student_id` = current user AND `course_id` = provided
3. Check enrollment for the subject:
   - If enrollment has `topic_id` → return that topic
   - If enrollment has `topic_id = null` → return ALL topics for the subject
4. Deduplicate results

**Response:**
```json
{
  "topics": [
    { "id": "uuid", "name": "Copyright Act" },
    { "id": "uuid", "name": "Patents Act" }
  ]
}
```

## Frontend Changes

### 1. Update `StartTestDialog`

**File:** `frontend/components/dialogs/start-test-dialog.tsx`

Change:
- `fetchSubjectsForCourse()` → Call new `enrollmentsApi.getEnrolledSubjects(courseId)` instead of `subjectsApi.getByCourse()`
- `fetchTopicsForSubject()` → Call new `enrollmentsApi.getEnrolledTopics(courseId, subjectId)` instead of `topicsApi.getBySubject()`

### 2. Add API Methods

**File:** `frontend/lib/api.ts`

```typescript
enrollmentsApi = {
  ...existing,
  getEnrolledSubjects: (courseId, token) => ...,
  getEnrolledTopics: (courseId, subjectId, token) => ...,
}
```

### 3. Add API Routes

**File:** `frontend/lib/api-routes.ts`

```typescript
ENROLLMENTS: {
  ...existing,
  ENROLLED_SUBJECTS: (courseId) => `${BASE_URL}/api/enrollment/subjects?course_id=${courseId}`,
  ENROLLED_TOPICS: (courseId, subjectId) => `${BASE_URL}/api/enrollment/topics?course_id=${courseId}&subject_id=${subjectId}`,
}
```
