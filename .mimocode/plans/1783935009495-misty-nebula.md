# Plan: Start Test Dialog & Socket Integration

## Overview
Add a "Start Test" button to My Tests page that opens a dialog for test configuration, then connects via Socket.IO for real-time test session.

---

## Step 1: Environment Variables
Add to `.env.local`:
```
NEXT_PUBLIC_SOCKET_URL=http://localhost:3005
```

## Step 2: Install Socket.IO Client
```bash
pnpm add socket.io-client
```

## Step 3: Update API Routes (`lib/api-routes.ts`)
Add:
```
TESTS.START: `${BASE_URL}/api/test/start`,
TESTS.SUBMIT: (id) => `${BASE_URL}/api/test/${id}/submit`,
TESTS.ABANDON: (id) => `${BASE_URL}/api/test/${id}/abandon`,
```

## Step 4: Update API Client (`lib/api.ts`)
Add methods:
- `testsApi.start(payload, token)` - POST /start
- `testsApi.submit(testId, token)` - POST /submit/:testId
- `testsApi.abandon(testId, token)` - POST /abandon/:testId
- `testsApi.getById(testId, token)` - GET /:testId

## Step 5: Create Socket Hook (`hooks/use-test-socket.ts`)
Hook that:
- Connects to Socket.IO with JWT token
- Emits `join_test`, `answer`, `skip`, `submit_test`, `heartbeat`
- Listens for `test_joined`, `answer_recorded`, `time_update`, `test_submitted`, `error`
- Sends heartbeat every 30s

## Step 6: Create Start Test Dialog (`components/dialogs/start-test-dialog.tsx`)
Dialog with:
- Duration selector (15, 20, 25, 30, 40, 45 min)
- Question limit input (with min/max validation based on duration)
- Course selection (from enrolled courses)
- Subject selection (filtered by course)
- Topic selection (filtered by subject)
- Dynamic selections list (add/remove)
- Start Test button

## Step 7: Update My Tests Page (`app/(dashboard)/dashboard/my-tests/page.tsx`)
- Add "Start Test" button
- Wire up dialog
- On start: call POST /start, then navigate to /take-test with testId

## Step 8: Update Take Test Page (`app/(test)/take-test/page.tsx`)
- Remove dummy data
- Fetch test session by ID from URL params
- Connect to socket via use-test-socket hook
- Wire up answer/skip/submit to socket events
- Handle time_update for real-time timer
- Handle test_submitted for completion
